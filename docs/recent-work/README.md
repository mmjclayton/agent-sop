# Recent Work — per-session entries

Each session's summary lives as its own file in this directory. The `## Recent Work (rollup)` section in the project's `CLAUDE.md` is a derived summary of these files, regenerated by `/update-sop` Step 8b on every run.

## Filename convention

```
YYYY-MM-DD_<agent-id>_<slug>.md
```

- Field separator: `_` (underscore)
- Within fields: `-` (hyphen) is allowed
- Slug: lowercase alphanumeric + hyphens, max ~50 chars, no underscores, no leading/trailing hyphen
- Agent-id: alphanumeric + hyphens, no underscores

Examples:
- `2026-04-19_solo_p43-batch-1-2-directory-structure.md`
- `2026-04-18_a7c3f2_fix-tonnage-edge-case.md`
- `2026-04-17_reviewer_refactor-auth-middleware.md`

## File format

```markdown
# [Session summary title]

**Date:** YYYY-MM-DD
**Agent:** <agent-id>
**Commits:** [hash, hash, ...]

[2-4 line summary of what shipped. Cross-reference Backlog P-numbers and build-plan batch numbers.]
```

## Archive

Entries older than 90 days move to `archive/` subdirectory. Never delete.
