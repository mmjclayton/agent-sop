P42 — Secondary-tracker reconciliation mechanism. Gap surfaced by hst-tracker: `/update-sop` treated `Backlog.md` as the sole work tracker; a project-specific `docs/audit-backlog-2026-04-18.md` kept 118 shipped audit items marked `[OPEN]` for a day because the SOP never told the agent to reconcile IDs that live elsewhere. Fixed at four points. (1) Core SOP Section 6 session-end checklist gained new step 3: reconcile any `.md` file in CLAUDE.md Key Documents that uses heading-level status tags. Total steps 7 → 8. (2) `/update-sop` Step 3b auto-detects tracker files via CLAUDE.md Key Documents scan (skip `Backlog.md`), extracts finding IDs from this session's commits, updates matching entries. Step 11 hard-blocks the commit if any referenced ID is still `[OPEN]`. (3) `/restart-sop` Step 4 gained an advisory drift guard — grep last 10 commits for finding IDs, flag still-`[OPEN]` matches. Advisory only (prior session may have had reason). (4) `[DEFERRED]` added to tag taxonomy as a distinct status from `[BLOCKED]` — blocked means waiting on external action, deferred means intentionally postponed with no blocker. Prevents stale `[OPEN]` items that were consciously pushed back. Heuristic design choice: auto-detect rather than explicit opt-in via config. Explicit opt-in recreates the original failure mode at a different level — user adds a new tracker file, forgets to register it, reconciliation silently skips it. Future `exclude` config field (already in follow-ups) becomes the escape hatch if false positives emerge. Net instruction count in core SOP: +1 (step 3 in Section 6), +1 (DEFERRED bullet in Section 8 with distinction). Still under 200 hard ceiling.
